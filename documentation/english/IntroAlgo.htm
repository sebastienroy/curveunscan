Curve recognition algorithms (continuous curves)

Curve following

From an initial point, chosen by the user, the border between the line and the curve background is followed to the left and to the right of the initial point.

According to the user choice, the upper(a) or the lower(b) border is used to follow the curve.

Threshold color

In the case of a monochromatic graphic, the black pixels are considered as part of the curve and the white pixels as the background. In the case of gray scale or colored graphics, the difference between curve and background is made as following :

  • A threshold color is chosen. The default values of its components are 70, on a scale from 0 (black) to 255 (white).
  • Every pixel having one of its components (red, green or blue) brighter than the threshold color related one is considered as being part of the background.

Stop condition

The curve is stopped when one of these conditions is satisfied :

  • The current pixel is outside the curve definition range
  • The end of the curve is reached
  • The followed curve crosses an other part of the same curve (case of several initial points)

Dashed Curves

CurveUnscan can perform recognition on dashed curves. If the "dashed curve" option is selected, the curve following, instead of stopping at the end of the dash, is extended to the next dashed and also from one to each other. The maximal distance between two dashes is specified by the user.

Initial point validity

The initial point is valid if the following conditions are satisfied:

  • It is part of a line (the value of each of its color components is less than the threshold one)
  • Its abscissa value belongs to the curve definition range
  • This point doesn't already belong to the followed curve. In the case of several initial points, the curves are build from left to right, following the initial points. If after the reconstruction of the whole curves (for instance in the case of changing the side to follow), an initial point belongs to the already built curve, this initial point is considered as redundant and is deleted.

Curves intersection

The curve intersections are handled as following (the different parts are generated from left to right) :

  • If the curves have no points with same abscissa (a), the whole curves are kept.
  • The curves are crossed (b). The right curve is stopped when its points belong to the left curve. The points of the left curve located to the right of this intersection are deleted.
  • If the curves are overlapping (c and d), the following of the right curve is stopped at the initial point of the left curve. The points of the left curve located to the right of the last point of the right curve are deleted.

Curve thickness compensation

Once the curve is built from the initial points, it is translated to the bottom or to the top, according to the side chosen to be followed.

Two algorithms are available in order to process this translation

Basic algorithm (b) : the points are translated from a constant value, equal to the half of the curve thickness. The curve thickness is specified by the user.

"Smart" algorithm (c) : the translation value depends on the curve thickness and on the rate of the curve. The number of points used to calculate the rate can be modified by the user.

The default algorithm is the "smart" one.
